# 數學能力小遊戲 開發文件

## 專案目標
在手機上執行的數學能力小遊戲。畫面上方顯示一道數學題目；下方顯示多個「公式或性質」選項，由使用者點選解題所需的性質，用以檢定或複習使用者對性質的掌握度。

## 主要玩法與流程
1. 顯示題目（單題模式）。
2. 顯示 3 至 6 個「公式/性質」選項。
3. 使用者點選 1 個或多個選項後送出。
4. 顯示是否正確，並列出正解所需的性質。
5. 記錄本題結果與耗時。
6. 進入下一題或重試。

## 核心功能需求
- 題庫與性質資料表
- 題目呈現與選項渲染
- 多選互動（點選切換）
- 送出作答與判分
- 即時回饋與解析
- 計分與進度紀錄
- 手機友善 UI（觸控、字體可讀、按鈕尺寸）

## 內容規格（資料結構）
### 題庫格式（建議）
每題包含：
- `id`: 唯一識別
- `stem`: 題目敘述
- `choices`: 需要顯示的性質選項清單（ID）
- `answers`: 正確性質選項清單（ID）
- `tags`: 題目主題
- `difficulty`: 1-5
- `explain`: 簡短解析

### 性質/公式格式（建議）
每項包含：
- `id`: 唯一識別
- `label`: 顯示文字
- `group`: 類別（代數、幾何、機率等）
- `short`: 短標籤（用於小按鈕）

## 判分規則（初版建議）
多選題：
- 完全命中 `answers` 且沒有多選 → 正確
- 少選或多選 → 錯誤
可在後續加入「部分得分」模式。

## 介面設計重點（手機）
- 上下兩區塊：題目區、選項區
- 選項以大型按鈕排列，適合拇指點按
- 送出/下一題固定位置
- 顯示作答狀態與正確性

## 技術實作方向（前端）
- 單頁面（HTML/CSS/JS）可先完成 MVP
- 資料以 JSON 管理題庫與性質
- 使用本地儲存記錄進度（localStorage）
- 導入 MathJax 顯示數學式（LaTeX 格式）
- 題庫索引獨立成 `index.json`，單題內容拆到 `questions/*.json`
- 讀取邏輯集中於 `data.js`，方便日後改為 DB 或 Google 試算表

## 數學式顯示（MathJax）
- 題目與性質文字可使用 LaTeX 語法，例如：`\\(a^2 + b^2 = c^2\\)`
- 建議使用 MathJax v3 CDN，載入後在題目/選項更新時呼叫 `MathJax.typeset()` 重新排版

## 資料讀取層（data.js）
- `data.js` 提供 `loadIndex()` 與 `loadQuestionByFile()`，集中處理資料來源
- 目前使用 `fetch(\"index.json\")` 與 `fetch(\"questions/*.json\")`
- 後續可替換為 DB API 或 Google 試算表 JSON

## 題庫索引格式（index.json）
- `properties`: 性質清單（全域共用）
- `questionIndex`: 題目索引清單，包含 `id`、`file`、`tags`、`difficulty`

## 里程碑（建議）
1. 題庫與性質資料結構確定
2. 基本 UI 與互動完成
3. 判分與回饋流程完成
4. 簡易統計與練習模式
5. 視覺優化與題庫擴充

## 後續可擴充功能
- 題目分類與篩選
- 難度自適應
- 錯題本
- 使用者成就與連續答對

## 修改需求紀錄
- 題庫改為索引 + 單題檔案（`index.json` + `questions/*.json`），避免一次載入全部資料
- 資料讀取邏輯集中於 `data.js`，可替換成 DB 或 Google 試算表
- 版面改為不滾動的全螢幕配置，確保手機上完整顯示
- 答案回饋改為彈窗顯示，點彈窗即可進下一題
